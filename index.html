ERP · SAP B1 · Odoo Industrias Bicicletas Milan S.A.

Unificación de ERPs y Gestión de Integridad de Datos Históricos

Administración simultánea de SAP Business One y Odoo.sh con más de 10 años de información histórica crítica, garantizando consistencia total entre sistemas.

✓ 0 errores de stock ✓ Sync bidireccional ✓ 10+ años migrados

⚡ Reto de Negocio

La empresa operaba con datos históricos de más de una década distribuidos entre SAP Business One y Odoo.sh, sin una fuente única de verdad. Los inventarios se actualizaban manualmente, generando discrepancias que afectaban las decisiones de compra, producción y logística. Las consultas de auditoría tardaban minutos, bloqueando los cierres contables.

🏗 Arquitectura & Stack

SAP Business One Odoo.sh Python SQL Server PostgreSQL Stored Procedures UDOs REST API

Capa de integración en Python como orquestador entre ambos ERPs y Magento. Stored Procedures y UDOs en SAP B1 para encapsular reglas de negocio. Módulos personalizados en Odoo.sh con herencia de modelos.

📈 Resultados

  • Sincronización bidireccional de inventarios en tiempo real entre SAP B1, Odoo y Magento.
  • Eliminación del ~100% de errores manuales de stock reportados mensualmente.
  • Consultas de auditoría reducidas de minutos a segundos con índices y vistas materializadas.
  • Integridad referencial garantizada sobre 10+ años de registros sin pérdida de datos.

> Detalle Técnico

PROBLEMA: La tabla MIL_RESUMEN_AUDITORIA acumulaba millones de registros
sin índices adecuados. Consultas de cierre mensual tardaban 4-7 minutos
bloqueando otras transacciones.

SOLUCIÓN TÉCNICA:
1. Análisis del plan de ejecución (EXPLAIN / SET STATISTICS IO ON)
   para identificar full table scans.
2. Índices compuestos sobre (FechaDocumento, TipoMovimiento, CodigoArticulo)
   — los campos más frecuentes en los WHERE de los reportes.
3. Particionamiento lógico por año fiscal con vistas indexadas (SQL Server)
   que pre-agregan totales por período.
4. SP_AUDITORIA_RESUMEN reescrito con CTEs en lugar de subconsultas
   correlacionadas, eliminando el N+1 implícito.
5. Triggers que rechazan inserciones con FechaDocumento fuera del período
   contable abierto, protegiendo la integridad temporal.

RESULTADO: Tiempo de consulta reducido de ~6 min → ~8 seg.
           Cierres contables sin bloqueos.

IA · Full-Stack · SEO Industrias Bicicletas Milan S.A.

Plataforma de Automatización de Marketing con IA Generativa

Aplicación web full-stack que integra modelos de lenguaje generativo para automatizar la creación de contenido SEO y reducir el trabajo manual del equipo de marketing en un 40%.

✓ -40% trabajo manual ✓ SEO automatizado ✓ Node.js + React

⚡ Reto de Negocio

El equipo de marketing dedicaba 3-4 horas diarias a tareas repetitivas: redacción de descripciones, meta-tags y adaptación de contenido para miles de SKUs. La escala del catálogo hacía inviable mantener contenido SEO optimizado de forma manual, resultando en fichas desactualizadas y pérdida de posicionamiento orgánico.

🏗 Arquitectura & Stack

Node.js React Gemini Pro API MySQL JWT Auth Prompt Engineering

SPA en React con backend Node.js como proxy seguro hacia la API generativa. Gestión de autenticación, rate limiting y estructuración de prompts en el servidor.

📈 Resultados

  • Reducción del ~40% en tiempo de trabajo manual del equipo de marketing.
  • Generación automatizada de descripciones SEO para miles de SKUs.
  • Pipeline completo: keyword research → generación → revisión → publicación.
  • Arquitectura extensible: compatible con OpenAI, Gemini y Claude.

> Detalle Técnico

ARQUITECTURA DE INTEGRACIÓN CON API GENERATIVA:

1. AUTENTICACIÓN SEGURA: La API Key nunca sale del servidor.
   El frontend usa JWT; el backend firma cada request hacia Gemini
   con la clave en variables de entorno.

2. PROMPT ENGINEERING ESTRUCTURADO: Plantillas parametrizadas por tipo
   de contenido (descripción, meta description, H1, bullet points).
   Cada plantilla inyecta contexto del producto + restricciones SEO.

3. PIPELINE: POST /api/content/generate
   → Validar payload → Construir prompt → Llamar generateContent()
   (temperatura 0.4) → Parsear y sanitizar → Persistir en DB
   con metadata (modelo, tokens, prompt_version) → Retornar con score SEO.

4. RESILIENCIA: Circuit breaker con reintentos exponenciales ante
   errores 429/503. Cola de trabajos para generaciones en lote.

Full-Stack · CRM · Laravel Soluciones Isavital SAS

CRM a Medida: Digitalización Total de la Gestión de Clientes

Diseño y desarrollo desde cero de un sistema CRM que digitalizó el 100% de la gestión de clientes, citas y mantenimientos operativos de una empresa de servicios.

✓ 100% digitalizado ✓ Retención mejorada ✓ Laravel + Livewire

⚡ Reto de Negocio

Isavital operaba con hojas de cálculo y registros en papel para gestionar clientes, citas y seguimiento postventa. La falta de trazabilidad generaba citas perdidas y una visión fragmentada del negocio. No existía ningún sistema que centralizara la información ni automatizara los recordatorios.

🏗 Arquitectura & Stack

Laravel Livewire PHP MySQL JavaScript Bootstrap Laravel Policies

Arquitectura MVC monolítica en Laravel con Livewire para reactividad sin SPA. Modelo relacional normalizado con sistema de roles y permisos granular.

📈 Resultados

  • Digitalización del 100% de la gestión de clientes, citas y mantenimientos.
  • Módulo de seguimiento postventa automatizado con notificaciones programadas.
  • Dashboard de métricas de retención y satisfacción en tiempo real.
  • Sistema de roles diferenciado para técnicos, administrativos y gerencia.
